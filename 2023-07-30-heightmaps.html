Heightmaps

2023, July 30

What’s a heightmap? Let’s ask the Wikipedia:

In computer graphics, a heightmap or heightfield is a raster image used mainly as Discrete Global Grid in secondary elevation modeling. Each pixel stores values, such as surface elevation data, for display in 3D computer graphics.

In summary they are neat and, in my opinion, quite simple to work with. So why not make a little program that parses heightmaps and makes a mesh?

Here are the structures we’ll be working with:

struct Heightmap {
    int width, height;
    std::vector<float> data;
};

struct HeightmapMesh {
    std::vector<vec3> vertices;
    std::vector<vec3> normals;
    std::vector<u32>  indices;
};

Alright. The first thing, is reading the heightmap. For that, we’re gonna use stb_image.h, since a heightmap is simply an image (at least in our case).

Heightmap load_heightmap(const std::string& path)
{
    int width, height, channels;
    int desired_channels = 1; // we need only one channel for a grayscale image

    u8* image = stbi_load(path.c_str(), &width, &height, &channels, desired_channels);

    std::vector<float> data;

    for (int i = 0; i < width * height; ++i) {
        // normalize values between [0, 1]; grayscale image's max value is 255.
        data.push_back(image[i] / 255.f);
    }

    stbi_image_free(image);

    return Heightmap{
        .width  = width,
        .height = height,
        .data   = data
    };
}

The next step is to build our mesh. To do that, we parse our heightmap to build a vector of vertices using the elevation as our y-coordinate parameter. For the normals, we sum the elevation of each neighbouring point and then we normalize it. After that, we simply generate the indices of each triangle of our mesh. It’s actually pretty easy, imagine you have a grid and simply begin tracing the triangles by joining the dots on a piece of paper.

HeightmapMesh heightmap_to_mesh(const Heightmap& heightmap)
{
    const auto  width  = heightmap.width;
    const auto  height = heightmap.height;
    const auto& data   = heightmap.data;

    HeightmapMesh result;

    for (int y = 0; y < height; ++y) {
        for (int x = 0; x < width; ++x) {
            // Get vertex

            result.vertices.push_back({ x, data[y * width + x], y });

            // Calculate the normal. Use the elevation data of each
            // adjacent vertex to do so.

            auto normal = vec3{ x, data[y * width + x], y };

            if (y != 0)
                normal += vec3{ x, data[(y-1) * width + x], y-1 };
            if (y != height-1)
                normal += vec3{ x, data[(y+1) * width + x], y+1 };
            if (x != 0)
                normal += vec3{ x-1, data[y * width + (x-1)], y };
            if (x != width-1)
                normal += vec3{ x+1, data[y * width + (x+1)], y };
            if (x != 0 && y != 0)
                normal += vec3{ x-1, data[(y-1) * width + (x-1)], y };
            if (x != width-1 && y != height-1)
                normal += vec3{ x+1, data[(y+1) * width + (x+1)], y };
            if (x != 0 && y != height-1)
                normal += vec3{ x-1, data[(y+1) * width + (x-1)], y+1 };
            if (x != width-1 && y != 0)
                normal += vec3{ x+1, data[(y-1) * width + (x+1)], y-1 };

            result.normals.push_back(normalize(normal));
        }
    }

    // Generate indices

    for (int y = 0; y < height-1; ++y) {
        for (int x = 0; x < width-1; ++x) {
            // Each cell in our heightmap can be seen as a quad (or
            // two triangles).

            // First triangle
            result.indices.push_back( y * width + x );
            result.indices.push_back( (y+1) * width + x );
            result.indices.push_back( y * width + (x+1) );

            // Second triangle
            result.indices.push_back( (y+1) * width + x );
            result.indices.push_back( (y+1) * width + (x+1) );
            result.indices.push_back( y * width + (x+1) );
        }
    }

    return result;
}

There’s a little caveat to keep in mind. If we apply some kind of scaling to our mesh then the normals need to be recalculated.

And that’s pretty much it.
